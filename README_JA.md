## 🌟 プロジェクト概要

本システムは、大学3年生の「知的制御」授業設計課題（手書き文字認識タスク）の要件に合わせて開発された、高性能な手書き文字 OCR 認識および自動修正システムです。**下層のカスタム CNN 畳み込みニューラルネットワーク構造 (`HandwrittenCNN`) を完全に維持**したまま、実用的な商用 OCR システム（QQ の文字抽出など）の洗練されたアルゴリズム思想を取り入れ、特に**「画像前処理」**と**「認識結果の修正」**の2つのプロセスにおいて大幅なアルゴリズムのアップグレードを行いました：
* **コア認識器**：既存の3層畳み込み層 `HandwrittenCNN` モデルを維持し、TTA（テスト時データ拡張）推論による複数サンプリングを採用。
* **影除去前処理**：**背景照明差分補正アルゴリズム**および**境界ボックス適応型クラスタリング・マージアルゴリズム**を導入。
* **インテリジェント修正**：**結合確率辞書デコーダ (Joint Probability Lexicon Decoder)** と**空間幾何学的校正（縦横比/相対高さ）**を搭載。
* **デバイスと操作**：PC 内蔵カメラと外部カメラの**ワンキー切り替え (`c` キー)**、および低信頼度文字に対する **Top-3 候補結果出力**をサポート。

## 🛠️ コアアルゴリズム設計（授業設計要件に完全対応）

### 1. データ収集と画像前処理（基本要件1）
実際のカメラでデモを行う際、環境の影（スマホや手の影など）によって、単純な二値化では黒い汚れが大量に発生します。
* **背景照明差分除去アルゴリズム**：大きなガウシアンフィルタ（$51 \times 51$）を用いて ROI 領域を平滑化し、背景の「照度マップ」（Illumination Map）を推定します。元のグレースケール画像を背景マップで除算（`cv2.divide`）することにより、均一な白い背景と鮮明な黒い筆跡を抽出し、影を完全に除去します。
* **CLAHE ＆ 適応型しきい値処理**：影を除去した画像に対して局所コントラスト適応型ヒストグラム均等化（CLAHE）および適応型しきい値処理（Adaptive Thresholding）を適用し、ノイズのない二値画像を得ます。

### 2. 特徴選択と文字分割（基本要件2）
* **適応型境界ボックスクラスタリング・マージ (Box Merging)**：
  - 輪郭の境界ボックスを自動でマージします。
  - **垂直方向マージ**：X軸の重なりが大きく、垂直距離がしきい値未満の場合、2つのボックスを1つに統合します（小文字 `i` や `j` の点と体が別々に分割される問題を解決）。
  - **水平方向マージ**：ペンでの筆記速度や細さによって途切れた筆跡（文字 `A` の横棒の隙間など）を自動で結合します。
* **EMNIST 基準重心位置合わせ**：マージしたボックスの重心（Image Moments）を計算し、$28 \times 28$ キャンバスの中心（$14, 14$）に平行移動させて CNN に送ります。

### 3. 認識結果の自動修正アルゴリズム（基本要件3 ＆ 最大の最適化点）
手書き文字で混同しやすい `oO0`, `1Il`, `2Zz`, `5Ss` は、$28 \times 28$ に正規化された画像単体では判別が非常に困難です。本システムは**結合確率辞書デコーダ**と**幾何学的制約**により全体校正を行います：
* **文脈環境の判定**：認識された文字配列が「英単語」か「数字/混合文字列」かを自動判別します。
* **結合確率辞書デコーダ (MAP)**：英単語モードの場合、単なる ArgMax（最大確率値の文字）によるデコードではなく、各位置における 62 次元の Softmax 確率分布と英単語辞書（常用1万語）の中で長さが一致するすべての単語との間で結合対数尤度を算出します：
  $$\text{Score}(W) = \sum_{i=1}^{N} \ln ( P(c_i.lower() | x_i) + P(c_i.upper() | x_i) )$$
  確率の積が最大の単語を出力します（例：CNN 原始出力 `he11O` $\rightarrow$ 辞書修正 `hello`）。
* **縦横比による `0` と `O` の区別**：アスペクト比 `w/h < 0.52` の文字は `0` に重みを加え、それ以外は `O/o` に割り振ります。
* **相対高さ比による大小文字の補正**：文字高さと行内最高文字との比率 $r = h_i / \max(H)$ を算出します。$r < 0.78$ かつ対称字（例：`Z`）と判定された場合、小文字 `z` に変換し、それ以外は元のケースを維持します。

## 🚀 実行および操作手順

### 1. リアルタイムカメラ認識システムの起動
```bash
python camera_detect.py
```
* **カメラの動的切り替え (`c` キー)**：
  - 起動時、システムは自動的にカメラインデックス `0`（通常は内蔵カメラ）をスキャンして開きます。
  - PC に複数のカメラが接続されている場合（内蔵カメラと外部 USB カメラなど）、画面がフォーカスされている状態で **`c` キー**を押すだけで、システムは実行を再起動することなく、利用可能なすべてのインデックスを巡回して動的に切り替えます。
* **分割と認識の実行（スペースキー）**：
  - 手書き文字が書かれた白紙を赤枠内に配置します（0.5mm 黑色ゲルペン推奨）。
  - `AI Vision` ウィンドウで、影が除去された鮮明な筆跡を確認できます。
  - **【スペースキー】**を押すと、文字分割とスペル修正が実行され、コンソールとメイン画面に「CNN原始予測」と「アルゴリズム修正後の最終結果」が即座に出力されます。
* **低信頼度文字の警告**：
  - 文字の信頼度が低い場合（確率が 80% 未満、または上位2つの差が 20% 未満）、黄色で `⚠️ 模糊字符警示`（上位3つの候補文字とその確率）が表示されます。
* **終了 (`q` キー)**：
  - **`q`** キーを押すと、キャプチャを解放し、すべてのウィンドウを安全に閉じます。

### 2. ビジュアルテスト評価の起動
```bash
python predict.py
```
- コンソールにシミュレーション単語（`hello`, `zoom`, `class`, `2026`, `10850`）の修正前後の比較レポートを表示します。
- その後、EMNIST テストセットから12枚の画像バッチをランダムにロードし、Matplotlib GUI でプロット表示します（緑：正解、赤：誤り、オレンジ：信頼度低）。Enter キーで次のバッチに進みます。
