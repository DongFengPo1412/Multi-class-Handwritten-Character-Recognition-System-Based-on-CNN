## 🌟 1. システム概要と主要技術指標

本システムは、エンドツーエンドかつ産業レベルの堅牢性を備えた、手書き文字認識（OCR）およびインテリジェントスペル修正システムです。カスタム畳み込みニューラルネットワーク（CNN）を用いて手書き文字の形状特徴を抽出し、フロントエンド画像取得、空間文字分割、ニューラルネットワーク推論最適化、ポストプロセッシング言語モデルスペル修正、および非同期人機インタラクションの5つの主要次元において高度なアルゴリズムのアップグレードとアーキテクチャ再構築を実現しています。

### コア機能の特徴
* **高堅牢性画像前処理**：環境の不均一な光や影の影響を排除する「背景照明差分補正アルゴリズム」を搭載。また、「自動コントラスト極性検出」により、白紙（明るい背景に暗い文字）と黒板（暗い背景に明るい文字）の双方に自動適応します。
* **高精度文字分割**：輪郭抽出前に形態学的閉演算を適用して筆跡の微細なかすれを物理的に橋接し、さらに適応型「繰り返し境界ボックスマージアルゴリズム」と重心モーメント位置合わせを組み合わせることで、連筆やかすれ、`i`・`j` などの複数パーツ文字の分割エラーを克服しています。
* **エンドツーエンドの推論加速**：3層畳み込みブロックからなる `HandwrittenCNN` を維持し、活性化関数に滑らかな SiLU (Swish)、重み初期化に Kaiming 基準を採用。推論時には TTA（テスト時データ拡張）マルチサンプリング融合とモデルウォームアップにより、初期推論遅延を排除しました。
* **確率的スペル自動修正**：英単語の文脈において、事後確率最大化（MAP）に基づく辞書対数尤度スコアリングを採用。アスペクト比の幾何検証や相対高さ比の解析と組み合わせることで、視覚的に混同しやすい手書き文字ペア（`0/O`、`1/I/l`、`2/Z`、大小文字など）をグローバルに自動修正します。
* **高フレームレート非同期GUI**：Tkinter を使用して応答性の高いシングルウィンドウダッシュボードを構築。`ThreadPoolExecutor` を用いたバックグラウンド処理により、カメラ映像の 30 FPS リアルタイム表示を維持しつつ、推論タスクと完璧に非同期デカップリングしています。

## 🛠️ 2. 数学的モデリングとアルゴリズムの革新

システム全体のデータ処理および計算パイプラインは以下の通りです：

```mermaid
graph TD
    A[カメラ/画像入力] --> B[背景光除去および極性自動検出]
    B --> C[形態学的閉演算および輪郭抽出]
    C --> D[繰り返しボックスマージおよび重心位置合わせ]
    D --> E[EMNIST 標準 $28 \times 28$ 文字テンソル]
    E --> F[CNN + TTA 推論予測]
    F --> G[低信頼度 Top-3 候補検出器]
    F --> H[辞書制約付き対数尤度スコアおよび幾何制約検証]
    H --> I[最終高精度テキスト出力]
```

### 2.1 画像前処理と環境への自動適応

#### 2.1.1 背景照明差分除去アルゴリズム
実環境下でカメラを使用する場合、手やスマートフォンの影が写り込み、単純な二値化では広範囲の黒い汚れが発生します。本システムは**背景照明差分除去アルゴリズム**によりこの課題を解決します。まず大核ガウシアンフィルタを用いて背景の局所的な照度マップ（Illumination Map）を推定し、除算処理を行います。

数学モデルは以下のように定義されます：
$$\text{Gray}_{\text{no\_shadow}}(x, y) = \min \left( \frac{\text{Gray}(x, y)}{G_{\sigma}(x, y) * \text{Gray}(x, y)} \times 255, 255 \right)$$
ここで $G_{\sigma}$ は標準偏差 $\sigma = 51$ のガウス平滑化カーネルを表します。この除算は OpenCV の `cv2.divide` による行列並列処理として実行され、影のない極めて鮮明な筆跡を抽出します。

#### 2.1.2 自動コントラスト極性検出
ユーザーによる手動切り替えを不要にするため、システムは二値化後に画像の外周境界ピクセルを判定します：
$$\Gamma = \text{Border}(\text{Thresh})$$
境界ピクセルの期待値 $E[\Gamma]$ を算出し、二値化後に $E[\Gamma] > 127$（明るい背景）である場合、EMNIST データセットのフォーマット（黒背景に白文字）に一致させるため、自動的に画像の反転処理を行います：
$$\text{Thresh}_{\text{input}}(x, y) = 255 - \text{Thresh}(x, y)$$
そうでない場合は極性を維持します。これにより、多様な紙や黒板等の筆記環境に対して100%自動で適応します。

### 2.2 文字分割および境界ボックスマージ

#### 2.2.1 形態学的閉演算による筆画橋接
極細ゲルペンでの記述やしきい値処理の限界により、筆画には微細なかする・切れが発生します。そのまま輪郭検出を行うと単一の文字が細切れに判定されてしまいます。そのため、輪郭検出の前に $2 \times 2$ の矩形構造要素 $S$ を用いて**閉演算（Closing Operation）**を適用します：
$$\text{Closed} = (\text{Thresh} \oplus S) \ominus S$$
この処理により、2ピクセル未満の隙間が自動的に結合され、文字分割の安定性が大幅に向上します。

#### 2.2.2 繰り返し境界ボックスマージ
従来の分割手法では単一方向の走査しか行わないため、隣接しないパーツの結合を見落としがちです。本システムは、適応型ルール `should_merge(box1, box2)` を用いた**繰り返しマージアルゴリズム**を採用し、ボックスの個数が収束するまで複数ラウンドにわたって処理を実行します。
2つの境界ボックス $B_1(x_1, y_1, w_1, h_1)$ と $B_2(x_2, y_2, w_2, h_2)$ のマージルールは以下の通りです：
1. **入れ子包含判定**：一方のボックスが他方のボックスにほぼ完全に包含されている場合（許容誤差 $\delta = 3$）、自動的に結合します。
2. **垂直方向の再グループ化（小文字 `i`、`j` などの点と体の結合）**：$B_1$ と $B_2$ のX軸上の重複投影比率 $O_x$ が $0.4$ を超え、かつ垂直方向の距離 $\Delta y$ が以下を満たす場合：
   $$\Delta y < \max(15, \min(h_1, h_2) \times 1.8)$$
   かつ、マージ後の高さが両者の最大高さの2.2倍を超えない場合、これらを1つの文字としてマージします。
3. **水平方向マージ（かすれ・断裂の修復）**：垂直方向の重複比率 $O_y > 0.5$ のとき、水平方向の間隔が $\Delta x \le 3$ ピクセル、または $\Delta x \le 6$ ピクセルでかつどちらか一方のボックス幅が極端に狭い（幅 $\le 5$ ピクセル、筆跡のかすれと判断される）場合、水平結合をトリガーします。

#### 2.2.3 重心モーメント位置合わせ（EMNIST規格への適合）
境界ボックス内での文字のズレが認識率に与える悪影響を排除するため、単純なアスペクト比変換ではなく、**画像物理モーメント（Image Moments）**に基づき高精度な重心移動を行います。
文字の二値画像スライスに対して、0次モーメント $M_{00}$ と1次モーメント $M_{10}, M_{01}$ を計算します：
$$M_{pq} = \sum_{x} \sum_{y} x^p y^q I(x, y)$$
重心座標は以下のように定義されます：
$$x_c = \frac{M_{10}}{M_{00}}, \quad y_c = \frac{M_{01}}{M_{00}}$$
文字画像を $20 \times 20$ ピクセルにリサイズした上で、標準の $28 \times 28$ キャンバス上に配置し、幾何中心 $(14, 14)$ への移動ベクトルを求めます：
$$[\Delta x_s, \Delta y_s] = [14.0 - x_c, 14.0 - y_c]$$
アフィン変換を用いて文字の重心位置を正確に $(14, 14)$ に合わせることで、手書きの位置ノイズを最小限に抑制します。

### 2.3 畳み込みニューラルネットワークと推論最適化

#### 2.3.1 HandwrittenCNN モデル構造
ネットワークは、以下のように3つの畳み込みブロックと全結合分類器で構成されています：

| ステージ | レイヤータイプ | 入力サイズ | 出力サイズ | パラメータ / 設定 |
| :--- | :--- | :--- | :--- | :--- |
| **ブロック 1** | Conv2d + BatchNorm2d + SiLU | $1 \times 28 \times 28$ | $32 \times 28 \times 28$ | $K=3$, $P=1$, $S=1$ |
| | MaxPool2d + Dropout2d | $32 \times 28 \times 28$ | $32 \times 14 \times 14$ | $Pool=2 \times 2$, $Drop=0.15$ |
| **ブロック 2** | Conv2d + BatchNorm2d + SiLU | $32 \times 14 \times 14$ | $64 \times 14 \times 14$ | $K=3$, $P=1$, $S=1$ |
| | MaxPool2d + Dropout2d | $64 \times 14 \times 14$ | $64 \times 7 \times 7$ | $Pool=2 \times 2$, $Drop=0.15$ |
| **ブロック 3** | Conv2d + BatchNorm2d + SiLU | $64 \times 7 \times 7$ | $128 \times 7 \times 7$ | $K=3$, $P=1$, $S=1$ (プーリングなし) |
| **全結合** | Flatten + Linear + SiLU + Dropout | 6272 | 512 | $Drop=0.5$ |
| **出力** | Linear | 512 | 62 | EMNIST 62クラスの分類 |

#### 2.3.2 Kaiming Normal 重み初期化
深層ネットワークにおける初期の勾配消失を防ぎ、学習速度を向上させるため、すべての畳み込み層に Kaiming（He）初期化を適用します：
$$W \sim \mathcal{N}\left(0, \sqrt{\frac{2}{\text{fan\_in}}}\right)$$
全結合層は平均 0、標準偏差 0.01 の正規分布で初期化し、バイアスはすべて 0 に初期化します。

#### 2.3.3 テスト時データ拡張（TTA, Test-Time Augmentation）推論
手書き特有の筆跡のブレによる単一サンプルの予測偏りを防ぐため、推論時に TTA マルチサンプリング融合機構を導入しています。
単一文字入力 $x$ に対して、システムは空間変形により以下の計11個のバリエーションを並列生成します：
* 平行移動：$\Delta x, \Delta y \in \{-1, 0, 1\}$（計9パターン）。
* わずかな回転変形：$\theta \in \{-5^{\circ}, 5^{\circ}\}$（計2パターン）。

これら11個のバリエーションを Batch 次元でマージしてモデルに送り、それぞれの出力に対する Softmax 確率の平均値を最終予測確率とします：
$$P_{\text{TTA}}(y \mid x) = \frac{1}{11} \sum_{k=1}^{11} P_{\text{model}}(y \mid \text{Transform}_k(x))$$
このテスト時アンサンブルによりノイズを効果的に平滑化し、識別器の汎化性能を大幅に向上させます。

### 2.4 多次元インテリジェントスペル修正モジュール

#### 2.4.1 事後確率最大化（MAP）に基づく辞書尤度スコアリング
視覚的に酷似した手書き文字（例：`hello` が CNN 単体で `he11o` や `he11O` と誤分類されるケース）は、純粋な視覚認識の限界です。本システムは、英単語の文字列と判定された場合に、常用英単語辞書 $D_L$（10,000語）の中から長さが一致する単語候補に対して尤度スコアリングを行います。

目標は、最も事後確率が高い単語 $W^*$ を決定することです：
$$W^* = \arg\max_{W \in D_L} \sum_{i=1}^{N} \ln \left( P(c_i^{\text{lower}} \mid x_i) + P(c_i^{\text{upper}} \mid x_i) \right)$$
ここで $P(c_i \mid x_i)$ は、各位置 $i$ で予測された TTA Softmax 確率値を表します。確率の積ではなく対数の和を計算することで、アンダーフローを防ぎ、精確な辞書校正が可能となります。

#### 2.4.2 縦横比による `0` と `O/o` の識別
形状が酷似した数字の `0` と英大文字の `O` について、境界ボックスのアスペクト比を用いた幾何学的先験ルールを導入しています：
$$\text{AR}_i = \frac{w_i}{h_i}$$
統計的に、手書き数字の `0` は縦長になりやすく、大文字の `O` は円に近くなります：
* $\text{AR}_i < 0.52$ の場合、数字の `0` に対する確率に重みを加算します。
* $\text{AR}_i \ge 0.52$ の場合、英文字 `O` / `o` への割り振りを優先します。

#### 2.4.3 相対高さ比に基づく大小文字補正
字形が対称な文字（例：`C/c`, `O/o`, `S/s`, `Z/z`）の誤認を防ぐため、文字列全体の中での相対高さ比を統計処理します：
$$r_i = \frac{h_i}{\max_{j=1}^N (h_j)}$$
対称文字において、相対的な高さ比が $r_i < 0.78$ の場合は小文字に、それ以外の場合は大文字に強制マッピングすることで、大小文字の乱れを自動で是正します。

## 📊 3. モデル学習と評価アセットの自動生成

### 3.1 ラベル平滑化（Label Smoothing）付きクロスエントロピー損失
モデルの学習には、過学習と過剰適合を防ぐためラベル平滑化損失関数を採用しています：
$$\mathcal{L}_{\text{LS}} = -(1 - \alpha) \log(p_c) - \frac{\alpha}{K} \sum_{k=1}^K \log(p_k)$$
ここで、平滑化係数 $\alpha = 0.1$、クラス数 $K = 62$ に設定されています。モデルが出力するロジット値が極端に大きくなるのを抑制し、手書きデータのラベルノイズ（類似文字の誤ラベリング等）に対する堅牢性を大きく高めています。

### 3.2 ハイパーパラメータ設定
* **オプティマイザ**：Adam（学習率 $\eta_0 = 10^{-3}$、L2正則化の Weight Decay = $10^{-4}$）を適用し、過学習を抑制。
* **学習率スケジューラ**：`ReduceLROnPlateau` を採用。検証損失が3エポック連続で停滞した場合、自動で学習率を半分に削減（Factor = 0.5）します。
* **早期終了（Early Stopping）**：検証損失の改善が7エポック連続で停滞した場合、自動でトレーニングを打ち切ります。

### 3.3 アカデミック評価データの自動書き出し (`checkpoints/` 内)
学習完了時に、以下の3つの高解像度評価チャートが自動的に書き出されます。プレゼンテーション用スライド（PPT）や技術レポートの作成に直接利用可能です：
1. **データ拡張サンプル (`data_augmentation_samples.png`)**：弾性歪みやアフィン回転を施した16枚の学習用データサンプル。
2. **学習収束曲線 (`training_curves.png`)**：Train/Val の損失（Loss）と精度（Accuracy）のエポック推移。
3. **62クラス混淆行列 (`confusion_matrix.png`)**：テストセットに対する詳細なエラー可視化ヒートマップ。`1/l/I` や `0/O` などの混同度合いを示し、後処理モジュール（`corrector.py`）の必要性を視覚的に証明します。

## ⚡ 4. UI設計と非同期並行処理

GUIには Tkinter を採用し、直感的なインスツルメントパネル風デザインを構築しています。

### 4.1 非同期スレッドプール（ThreadPoolExecutor）の統合
* **課題**：画像認識処理やネットワーク API（百度）への同期型リクエストを GUI の描画と同一スレッドで処理すると、認識ボタン押下時にカメラ映像が数秒間フリーズする現象が発生します。
* **解決策**：`ThreadPoolExecutor` を用いて計算処理をGUIスレッドからデカップリングしました。
  * **メインスレッド**：$30\text{ms}$ 間隔でカメラフレームを読み込み、背景光除去と二値化処理を適用してリアルタイムで $30\text{ FPS}$ のプレビュー描画を継続。
  * **ワーカースレッド**：ユーザーがスペースキーを押すと、現在の関心領域（ROI）をコピーしてバックグラウンドで CNN + TTA 推論と API リクエストを処理。結果が得られ次第、コールバックを介してUIに反映するため、描画のカクつきは一切発生しません。

### 4.2 双方向状態マシン（Live / Freeze モード）
ユーザーが検出時の特徴を静止状態で詳細に観察できるよう、双方向の状態遷移を定義しています：
* **LIVE モード**：右上のステータスバッジが緑色の `LIVE` になり、映像と二値化画像が 30 FPS でリアルタイムに更新されます。
* **FREEZE モード**：スペースキーを押すと、ステータスが `FREEZE`（オレンジ色バッジ）に遷移し、映像フレームがロックされます。このとき、分割された各文字領域が**青色 (Cyan)**の枠と番号（1, 2, 3...）で画像上にオーバーレイ表示されます。
* **解除方法**：`Space`, `Enter`, `Esc` キーまたは `Resume` ボタンのクリックで即座に LIVE モードに戻ります。

```bash
pip install -r requirements.txt

$env:BAIDU_OCR_API_KEY = "Your_Baidu_API_Key"
$env:BAIDU_OCR_SECRET_KEY = "Your_Baidu_Secret_Key"
```

### 6.2 GUI ダッシュボードの起動
以下のコマンドでアプリケーションを起動します：
```bash
python desktop_app.py
```
* **カメラの切り替え (`C` キー)**：複数のカメラが接続されている場合、ウィンドウにフォーカスを当てて **`C` キー** を押すと、カメラインデックスが動的に切り替わります。
* **文字の切り出し・認識 (Space キー)**：手書きの文字を中央の赤枠内に配置して **【Space】** を押すと、画面が定格（FREEZE）し、青枠とインデックス番号で切り出された文字が表示されます。
* 右側の結果表示パネルには、上から順に以下が表示されます：
  1. ローカル CNN の生（ArgMax）予測結果
  2. 辞書モデルと幾何検証を経た修正（Corrected）結果
  3. 百度クラウド OCR 予測結果（有効時）
  4. リアルタイム適応型二値化追跡マップ
* もう一度 **Space / Enter / Esc** を押すと、定格が解除されてリアルタイムプレビューに戻ります。
* **`B` キー** で、クラウド OCR 機能の有効・無効をいつでも切り替えられます。

### 6.3 オフラインスペル修正テストの実行
スペル修正アルゴリズムの予測性能をコンソールで検証する場合：
```bash
python predict.py
```
`he11o` や `z00m` などの混同スペルが `hello`、`zoom` に自動補正される過程がテストレポートとして出力され、EMNISTデータを用いた可視化評価ウィンドウが順次表示されます。
