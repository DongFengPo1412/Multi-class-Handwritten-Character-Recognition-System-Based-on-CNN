# 基于 CNN 的多类别手写字符识别系统
(Multi-class Handwritten Character Recognition System Based on CNN)

这是一个基于 **PyTorch 深度学习框架**与 **OpenCV 图像处理技术**实现的高效、鲁棒的多类别手写字符识别系统。系统支持 62 类手写字符（包含 10 个数字、26 个大写英文字母和 26 个小写英文字母）的精准识别，并针对 **0.5mm 签字笔细笔迹**以及**真实摄像头拍摄环境**进行了全链路的算法优化与硬件加速。

---

## 🌟 系统亮点与关键技术

### 1. 轻量高效的网络架构 (`HandwrittenCNN`)
- **模型结构**：精心设计了 3 层卷积块（卷积层 + 批归一化 BatchNorm2d + 激活函数 ReLU + 最大池化 MaxPool2d / 空间丢弃 Dropout2d），尾部接入双层全连接层。
- **高代表性特征**：在第三个卷积块中取消了池化操作，以保留丰富的 $7 \times 7$ 局部拓扑特征，最终展开为 6272 维向量输入全连接层，保证了轻量化的同时拥有强劲的拟合能力。

### 2. 强泛化性的全维度数据增强 (`get_dataloaders`)
针对手写纸张倾斜、字符大小不一、光照复杂等真实应用场景，在训练集中应用了超强数据增强流水线：
- **空间变换**：利用随机仿射变换（旋转 15°、平移 12%、缩放 0.8-1.2 倍、剪切 12°）模拟书写倾斜和距离变化。
- **透视畸变**：通过随机透视变换（RandomPerspective，概率 40%）模拟相机非正对纸张时的视角偏差。
- **褶皱颤抖**：应用随机弹性变换（ElasticTransform，概率 20%）模拟纸张不平整与笔画颤抖。
- **光影与模糊**：引入颜色抖动（ColorJitter，亮度/对比度抖动 0.3）和高斯模糊（GaussianBlur，概率 20%）对抗灯光阴影与对焦不准。
- **拓扑泛化**：引入随机擦除（RandomErasing，概率 15%），迫使模型学习字符的整体拓扑结构而非局部笔画。

### 3. 测试时增强推理 (TTA, Test-Time Augmentation)
- 在 `predict.py` 与 `camera_detect.py` 的推理过程中，单张图像在送入模型前会被变换为 **11 个变体**（包含原始图像、8 个方向的 1 像素微小平移、以及左右 5° 的小角度旋转）。
- 将这 11 个变体的预测概率分布进行平均，作为最终的输出置信度，极大地增强了系统对于字符位置偏移和倾斜的鲁棒性。

### 4. 易混淆字符的逻辑修正机制 (Correction Logic)
- 针对 62 类字符中存在天然字形歧义的字符（例如 `0` 与大写 `O`、小写 `o`；`1` 与大写 `I`、小写 `l` 等），系统会在推理后检查 Top-2 预测。
- 当两个易混淆候选字符的概率差值小于 0.2 时，判定为“歧义状态（Ambiguous）”，并在可视化界面中用黄色警示，提示进行人工二次确认，从而规避硬分类带来的误导。

### 5. 实时摄像头图像处理流水线 (`camera_detect.py`)
- **区域定位**：在画面中央设立 350x350 的检测窗口（ROI）。
- **光影均衡化**：采用 **CLAHE (自适应直方图均衡化)** 配合中值与高斯双重滤波，能够彻底消除浅色纸张在不同灯光环境下的阴影死角。
- **动态笔画加粗**：根据分割出字符的像素密度，动态执行膨胀（Dilate）或腐蚀（Erode）操作。若检测到是 0.5mm 签字笔产生的细笔画（密度低），则自动进行像素加粗，使笔迹贴近 EMNIST 官方标准。
- **质心精确对齐**：使用图像矩（Moments）计算提取字符的物理重心，将其平移对齐至 $28 \times 28$ 规范画布的中心（$14, 14$）。
- **多字符顺序识别**：提取窗口内所有有效轮廓后，过滤噪点并按从左至右的 $X$ 坐标自动排序，实现单次拍摄多个手写字符的流式识别。

### 6. 极致硬件加速 (RTX 4060 GPU)
- **混合精度训练**：在 `train.py` 中引入 `torch.amp.autocast('cuda')` 及 `GradScaler`，实现 FP16 与 FP32 混合精度训练，充分释放显卡 Tensor Core 算力。
- **IO 加速**：利用 PyTorch DataLoader 的 `pin_memory=True`（锁页内存加速主机到设备的数据拷贝）和 `persistent_workers=True`（常驻多进程减少 Epoch 间的多进程销毁重建开销）来榨干硬件读写极限。

## 🚀 快速开始

### 1. 安装依赖环境
确保已配置好 Python 环境（推荐使用 Conda 或虚拟环境），然后安装以下依赖：
```bash
pip install torch torchvision numpy opencv-python matplotlib
```

### 2. 训练模型
运行 `train.py` 开始训练网络。该脚本会自动从 PyTorch 官方数据源下载 EMNIST ByClass 数据集到 `data/` 目录。
```bash
python train.py
```
- **配置参数**：默认最大训练 25 轮。
- **负反馈调度**：若连续 3 轮验证集 Loss 不降，则自动将学习率减半。
- **早停机制**：若连续 7 轮验证集准确率未提升，将提前终止训练以防止过拟合。
- **最优权重**：会自动保存在 `checkpoints/emnist_model.pth` 中。

### 3. 可视化测试
通过 `predict.py` 验证模型在完全未见过的测试集上的预测表现。它会使用交互式窗口展示识别结果：
```bash
python predict.py
```
- **颜色标识**：
  - <font color="green">**绿色 (Green)**</font>：预测正确且高置信度。
  - <font color="orange">**橙色 (Orange)**</font>：预测正确但处于歧义区间（例如 0 和 O 的概率极近），提示需人工确认。
  - <font color="red">**红色 (Red)**</font>：预测错误。
- 按 **回车键** 切换下一批图像，输入 **`q`** 退出测试。

### 4. 实时摄像头字符识别
将手写纸张对准摄像头，运行 `camera_detect.py`：
```bash
python camera_detect.py
```
- **操作指南**：
  1. 将手写好的字符（推荐使用 0.5mm 黑色签字笔书写在白色作业纸上）置于红色检测框内。
  2. 可以通过 `Internal AI Vision` 窗口实时观察 AI 视角的二值化去噪和笔画调节效果。
  3. 按下 **[空格键]**，系统将自动分割框内的所有字符，并按从左至右的顺序在命令行及窗口顶部显示识别出的字符序列。
  4. 按下 **`q`** 键退出系统。
